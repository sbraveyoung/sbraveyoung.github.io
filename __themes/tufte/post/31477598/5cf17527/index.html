AI 入门到精通 00 · 导读与学习路线

〇、这套系列是什么

一份面向完全零基础的 AI 学习路线图，目标是把你从「听过 ChatGPT」带到「能独立训练模型 / 写 Agent / 跟进前沿论文」。

怎么用这份路线

  • 顺序：00 → 10 是推荐路径，但不是硬性顺序。如果你已经有编程或数学基础，可以跳过对应章节。
  • 节奏：全职学习 6-9 个月可达到「能独立做 AI 项目」的水平；业余学习按 2 倍时间估算。
  • 原则：学习量 = 阅读 × 实践 × 复盘。每一章都给出「动手实践」，不要跳过。
  • 检验：每章末尾的「通关标志」是判断是否可以进入下一章的硬指标。完不成就别往后走。

路线总览

# 章节 主题 估时（全职）
00 导读与心态 为什么学、怎么学、避坑 1 天
01 数学基础 线代 / 微积分 / 概率 / 优化 6-8 周
02 编程能力 Python / NumPy / PyTorch / 工程化 4-6 周
03 机器学习基础 经典 ML 算法与方法论 4-6 周
04 深度学习基础 NN / CNN / RNN / Transformer 6-8 周
05 大语言模型 LLM 架构、训练范式、评测 4-6 周
06 AI 训练实战 数据 / 分布式 / 调参 / 监控 4-6 周
07 AI 推理与部署 量化 / 加速 / 服务化 3-4 周
08 AI Agent 工具调用 / 规划 / 多 Agent 持续
09 多模态与生成 CLIP / Diffusion / 视频生成 4-6 周
10 前沿方向 推理增强 / 对齐 / Embodied 持续

一些必须先纠正的误解

  • "AI = 调 OpenAI API"：那是 使用 AI，不是 做 AI。两者职业天花板差一个数量级。
  • "必须读完数学才能开始"：错。数学和代码是螺旋上升的，遇到不懂的回头补即可，不要试图一次学完。
  • "看视频 = 学会了"：看完不写一行代码 = 没学。所有材料都附了 hands-on，跳过 = 白学。
  • "模型越大越好"：工程上 90% 的问题用小模型 + 好数据就能解决。先学会做小模型，再谈 scaling。

一、这套教程要解决什么问题

市面上 AI 教程的两个极端：要么是「30 天精通深度学习」式的速成，看完只会调 API；要么是「读完《Deep Learning》(花书) 再来」，门槛高到 90% 的人放弃。

本路线的定位是：让一个完全不懂 AI 的人，按部就班走完后能独立完成「数据 → 训练 → 部署 → 改进」的完整闭环，并且具备追前沿论文的能力。

二、AI 这个领域到底有什么

按职业分工大致是这几条线，新手不需要立刻选定，但要心里有数：

方向 在干什么 必备技能
算法研究 (Research) 提新方法、写论文 数学、论文阅读、PyTorch、实验设计
算法工程 (Applied) 把模型落地到业务 DL 基础、工程化、调参、领域知识
基础设施 (Infra) 训练 / 推理框架、加速 CUDA、分布式系统、编译器
数据 (Data) 数据采集、清洗、标注体系 SQL、Spark、数据治理、领域知识
产品 (Product) AI 产品设计、PMF 用户洞察、对模型能力边界的直觉
Agent / 应用 LLM 编排、工具集成、Workflow 提示工程、API 集成、领域 know-how

新手友好度排序大致是：Agent / 应用 > 算法工程 > Infra > 数据 > 算法研究。但天花板顺序大致反过来。

三、学习方法论

  1. 费曼学习法：学完一章，写一篇博客（或对着 ChatGPT 讲一遍），讲不清就是没懂。
  2. 20% 学习 + 80% 实践。所有「知识点」必须配「能跑起来的代码」。
  3. 写下你的问题清单。学每个概念前先问：它解决什么问题？为什么之前的方法不行？它的局限是什么？
  4. 维护一个 paper / 项目仓库：把读过的论文、跑过的代码、失败的实验都记下来。这就是你的「肌肉记忆」。
  5. 不要追新。基础没打好就追 SOTA 是浪费时间。GPT-4 的根基还是 2017 年的 Transformer。

四、新手最常见的坑

  • 环境配置劝退：先在 Google Colab 或 Kaggle Notebooks 上跑代码，等熟练了再折腾本地 GPU / CUDA。
  • 数学焦虑：见到公式跳过 → 永远学不会。数学不会就停下来补，不要硬撑。
  • 只看不写：60% 的"学过 ML"的人写不出 NumPy 实现的线性回归。
  • 盲信教程：教程是给你思路，不要逐行抄。每一行都要问"为什么是这样"。
  • 追求完美：你写的第一个模型一定是垃圾，跑通就是胜利。

五、配套环境准备

最低配置（够走完 01-05）：

  • 硬件：随便一台能开浏览器的电脑 + Google Colab 免费 T4 GPU 即可。
  • 账号：Google 账号、GitHub 账号、Hugging Face 账号、Kaggle 账号、OpenAI / Anthropic / DeepSeek 任一 API key（小额充值 5-10 美元够用很久）。
  • 软件：VS Code + Python 扩展、Git、一个能科学上网的方案（论文 / Hugging Face / GitHub 在国内访问偶有不便）。

进阶配置（06 之后）：

  • 一张本地 GPU（RTX 3090 / 4090 起，24G 显存能跑 7B 模型 LoRA），或者按需租用 AutoDL / RunPod / Lambda Labs 等。
  • Linux 操作系统（双系统 / WSL2 / 云主机均可）。

六、参考资料

入门期通读，之后反复翻阅：

  • 书：吴军《数学之美》—— 不教技术，但帮你建立"AI 是怎么思考问题的"的直觉。
  • 书：《动手学深度学习》(d2l.ai 中文版免费) —— 后面整个路线都会引用。
  • 课：吴恩达《Machine Learning Specialization》(Coursera 新版) —— 入门门槛最低的系统课。
  • 视频：3Blue1Brown 的神经网络系列 —— 看一遍胜过读十篇博客。
  • 阅读：Andrej Karpathy 的 Neural Networks: Zero to Hero —— 后面会反复推荐。

七、学习资源元清单（贯穿整个路线）

  • 论文：arXiv 的 cs.LG / cs.CL / cs.CV
  • 代码：GitHub 上 trending、HuggingFace models & datasets
  • 课程聚合：Deep Learning Specialization (Coursera)、fast.ai、Stanford CS 系列
  • 新闻 / 解读：The Batch (deeplearning.ai)、Import AI、Hugging Face Daily Papers
  • 中文社区：知乎 AI / LLM 话题、机器之心、PaperWeekly

资源会过时，但路径不会。优先按本路线走，资源没了就找替代品。

八、动手实践（在开始 01 之前完成）

  1. 注册上面列出的全部账号。
  2. 在 Colab 上跑一个 "Hello, GPU" notebook：用 torch.cuda.is_available() 验证能用 GPU，并跑一个最简单的 torch.nn.Linear 前向传播。
  3. 用任一 LLM API（推荐 DeepSeek 或 OpenAI）写一个 20 行 Python 脚本：输入一段中文，输出英文翻译。
  4. 在 GitHub 上建一个 ai-learning repo，把今天的代码 commit 上去。这个 repo 会陪你走完整个路线。

通关标志

  • 能向一个完全不懂 AI 的朋友解释清楚「机器学习」「深度学习」「大模型」的区别。
  • 你知道自己未来感兴趣的方向（不要紧，后面可以改）。
  • 学习仓库已建好，能跑通 GPU。
  • 调通过至少一次 LLM API。

完成后进入 01. 数学基础。
